# 102 用户流程文档

## 1. 角色

- 字幕编辑者：主要用户，负责导入、生成、调整、导出
- 剪辑协作者：需要拿到 SRT 或 FCPXML 结果的人
- 自动化操作者：偏好目录监听，减少重复手工操作的人

## 2. 主流程总览

1. 首次进入与导入
2. 字幕生成
3. AI 校对
4. 字幕编辑
5. 导出交付
6. 目录监听自动处理
7. 设置配置与验证

## 3. 流程详情

### 3.1 首次进入与导入

目标：让用户快速开始一次字幕任务。

步骤：

1. 用户打开应用。
2. 用户看到明确的导入入口、最近项目、当前默认配置摘要。
3. 用户通过点击打开文件或拖入素材开始任务。
4. 系统识别是音频素材还是 SRT 文件。
5. 如果是 SRT，直接进入编辑；如果是音频，进入转写流水线；视频文件提示不支持。

关键输出：

- 当前处理文件
- 进入哪条后续路径
- 需要时可前往设置

### 3.2 字幕生成

目标：把原始素材转成可编辑的字幕片段。

步骤：

1. 系统准备文件。
2. 系统调用选定转写引擎（只做听写 / 协议适配）。
3. 引擎结果归一为带时间词元（无词级时间则估算）。
4. 公共分段器按统一规则切分 / 合并为字幕片段。
5. 页面展示当前步骤和整体进度。
6. 成功后进入编辑页。
7. 失败时给出可理解的原因和重试路径。

关键反馈：

- 当前步骤
- 是否完成
- 失败原因
- 当前引擎（切换引擎不改变分段规则来源）

### 3.3 AI 校对

目标：在不替代人工审核的前提下，提高识别文本质量。

步骤：

1. 用户在设置中开启 AI 校对。
2. 转写完成后，系统自动进入校对阶段。
3. 校对完成后把结果返回到字幕片段中。
4. 用户继续人工确认和修改。

关键规则：

- AI 校对可选
- 校对失败不应毁掉已有转写结果

### 3.4 字幕编辑

目标：让用户高效完成人工微调。

步骤：

1. 用户进入编辑工作台。
2. 用户查看当前选中字幕的文本和时间。
3. 用户逐条修改文本、入点、出点。
4. 用户按需插入空字幕、合并相邻字幕。
5. 用户回放音频并校准时间点。
6. 用户完成修改后准备导出。

关键规则：

- 当前选中项要清楚
- 编辑动作必须轻量
- 不让低频操作干扰主路径

### 3.5 导出交付

目标：把编辑结果转换成外部可用文件。

步骤：

1. 用户点击导出。
2. 系统先保存字幕文本结果。
3. 系统生成 FCPXML 等交付文件。
4. 如果是 FCP 工作流，系统可直接打开交付文件。

关键结果：

- 输出文件路径明确
- 输出文件格式稳定
- 与 Final Cut Pro 工作流衔接自然

### 3.6 目录监听自动处理

目标：减少 FCP 工作流中的重复手工操作。

步骤：

1. 用户在设置中指定监听目录。
2. 用户启动监听。
3. 系统发现新素材后自动转写。
4. 根据模式决定自动导出或进入人工校对。
5. 完成后产出可交付文件，并反馈处理结果。

关键规则：

- 监听是增强模式，不是默认模式
- 自动化过程必须能被中止和观察

### 3.7 设置配置与验证

目标：把所有配置集中到独立页面中处理。

步骤：

1. 用户打开设置。
2. 调整转写、校对、样式、导出、监听参数。
3. 通过测试能力验证配置是否可用。
4. 保存并回到主流程。

关键规则：

- 设置不占据主编辑页面
- 配置是否生效应可验证
