# 103 产品架构文档

## 1. 架构目标

本文件描述的是产品层架构，不绑定当前代码实现。

目标：

- 保证主流程清晰
- 保证设置、编辑、导出、监听职责分离
- 为后续重做提供稳定的模块边界

## 2. 推荐分层

### 2.1 交互层

负责用户看到和操作的界面：

- 首页 / 导入入口
- 转写进度页
- 字幕编辑页
- 设置页
- 提示与状态反馈

### 2.2 流程层

负责把多步骤任务串起来：

- 导入路由
- 转写流程
- 校对流程
- 导出流程
- 目录监听流程

### 2.3 领域层

负责真正稳定的业务对象和规则：

- 带时间词元（转写中间结果）
- 字幕片段（可编辑最终单元）
- 公共分段规则（标点、字数、时长、停顿、英文边界）
- 项目状态
- 导出配置
- 字幕样式
- 最近项目

### 2.4 集成层

负责和外部能力对接：

- 本地转写引擎（Whisper / FunASR SenseVoice）
- Apple Speech
- 云端 ASR
- 云端 LLM
- 文件系统
- Final Cut Pro

集成层只负责「听写」与协议适配，不各自实现一套字幕切分策略。

## 3. 核心边界

- 设置中心只负责配置，不承载主编辑逻辑
- 编辑工作台只处理字幕和播放，不承担复杂配置
- 导出只依赖已经确认过的字幕数据和导出参数
- 目录监听属于旁路工作流，不污染主编辑流程
- 日志与测试能力属于支撑系统，不直接出现在主任务路径中
- **转写与分段解耦**：各 ASR 引擎只产出带时间的原始词元 / 粗段落；字幕切分、合并、去重叠由公共分段器统一完成
- 单条字幕最大字数等分段参数作用于公共分段器，而不是分别塞进各个引擎

## 4. 字幕生成两阶段数据流

旧逻辑容易变成「三种转写方式各自切句」；当前产品边界要求：

```
音频
  → 引擎适配（Apple / 本地 Whisper / 本地 FunASR / 云端 ASR）
  → 带时间词元 [SubtitleWord]  （或无可靠词时间时的粗 segment，再估算词元）
  → 公共分段器（TimedSubtitleSegmenter）
  → 可编辑字幕片段 [SubtitleSegment]
  → （可选）AI 校对
  → 编辑 / 导出
```

步骤说明：

1. 用户导入素材，系统生成项目上下文。
2. 按设置选择转写引擎；引擎只负责识别与协议细节（本地 CLI、系统 Speech、HTTP/异步任务等）。
3. 引擎结果归一为带时间词元：
   - 有真实词级时间：直接使用（Whisper DTW token、DashScope words 等）
   - 只有粗段落：先估算词元时间，再进入同一分段器（本地 FunASR / SenseVoice 一期属于此路径）
4. 公共分段器按强/软标点、最大字数、最大/优选时长、停顿间隙、英文不拆词、时间不重叠等规则生成字幕片段。
5. 校对服务可选地修正字幕文本。
6. 编辑工作台消费并修改字幕片段。
7. 导出流程基于当前字幕片段输出交付文件。
8. 监听流程复用同一套「转写 → 公共分段 → 校对 → 导出」能力。

## 5. 后续重做约束

- 不要求保留当前 View / Service 切分方式
- 不要求保留现有脚本命名
- 只要求保留文档中定义的功能边界、输入输出和规则
- **必须保留「转写管转写、分段管分段」边界**：新增 ASR 源时只做词元适配，禁止再复制一套切句逻辑

## 6. 智能字幕跨服务关系

```text
StoreKit / Billing ──履约──► Model API钱包
                              ▲
SubForge ──业务请求+官方Key───┘
   │
   └──临时上传策略──► 阿里中国区临时OSS

Model API ──中国区Key──► DashScope ASR / AI校对
```

- SubForge不保存或接收DashScope永久Key。
- 客户端只提交`processingRegion=china`，不提交Base URL、Workspace或模型。
- 官方钱包Key保存在macOS Keychain；用户自定义Provider Key与官方智能服务隔离。
