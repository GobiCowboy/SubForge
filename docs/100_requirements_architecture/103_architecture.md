# 103 产品架构文档

## 1. 架构目标

本文件描述的是产品层架构，不绑定当前代码实现。

目标：

- 保证主流程清晰
- 保证设置、编辑、导出、监听职责分离
- 为后续重做提供稳定的模块边界

## 2. 推荐分层

### 2.1 交互层

负责用户看到和操作的界面：

- 首页 / 导入入口
- 转写进度页
- 字幕编辑页
- 设置页
- 提示与状态反馈

### 2.2 流程层

负责把多步骤任务串起来：

- 导入路由
- 转写流程
- 校对流程
- 导出流程
- 目录监听流程

### 2.3 领域层

负责真正稳定的业务对象和规则：

- 字幕片段
- 项目状态
- 导出配置
- 字幕样式
- 最近项目

### 2.4 集成层

负责和外部能力对接：

- 本地转写引擎
- Apple Speech
- 云端 ASR
- 云端 LLM
- 文件系统
- Final Cut Pro

## 3. 核心边界

- 设置中心只负责配置，不承载主编辑逻辑
- 编辑工作台只处理字幕和播放，不承担复杂配置
- 导出只依赖已经确认过的字幕数据和导出参数
- 目录监听属于旁路工作流，不污染主编辑流程
- 日志与测试能力属于支撑系统，不直接出现在主任务路径中

## 4. 数据流

1. 用户导入素材。
2. 系统生成项目上下文。
3. 转写服务输出字幕片段。
4. 校对服务可选地修正字幕片段。
5. 编辑工作台消费并修改字幕片段。
6. 导出流程基于当前字幕片段输出交付文件。
7. 监听流程重复使用同一套转写、校对、导出能力。

## 5. 后续重做约束

- 不要求保留当前 View / Service 切分方式
- 不要求保留现有脚本命名
- 只要求保留文档中定义的功能边界、输入输出和规则
