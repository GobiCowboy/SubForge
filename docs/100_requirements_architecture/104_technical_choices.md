# 104 技术选择文档

## 1. 技术选择目标

这里只记录对产品形态有影响的技术决策，不记录临时代码实现。

## 2. 已锁定的产品级选择

| 主题 | 选择 | 原因 |
|------|------|------|
| 运行平台 | macOS 桌面应用 | 目标用户明确，且要衔接 Final Cut Pro |
| 工作模式 | 本地优先 | 降低云依赖和等待成本 |
| 主输入 | 音频、SRT | 当前只处理音频转写和现成字幕编辑，不支持 MP4、MOV、MKV 等视频文件 |
| 主输出 | SRT、FCPXML | 满足通用字幕交付与 FCP 工作流 |
| 配置方式 | 本地持久化 | 单用户桌面工具更简单直接 |
| 智能增强 | AI 校对可选 | 提升质量但不阻塞主流程 |
| 自动化能力 | 目录监听可选 | 服务进阶工作流，不影响基础使用 |

## 3. 引擎策略

- 转写引擎支持本地与云端并存（Apple 语音 / 本地 Whisper / 本地 FunASR SenseVoice / 云端 ASR）
- 默认优先本地方案
- 中日韩英多语本地优先可选 **SenseVoice GGUF**（`llama-funasr-sensevoice`）；中文词级时间更强时可继续用 Whisper DTW 或云端 words
- 当本地不可用或质量不足时，允许切到云端
- AI 校对是增强能力，不应成为唯一通路

## 3.1 转写与分段职责（已锁定）

| 职责 | 归属 | 说明 |
|------|------|------|
| 听写 / 协议 | 各引擎适配器 | 调用 Apple Speech、whisper-cli、云端 HTTP/异步任务；解析厂商原始结果 |
| 词元归一 | 引擎出口 | 统一产出带时间的词元；无词级时间时走估算路径，不得直接各写一套切句 |
| 切分 / 合并 | 公共分段器 | 标点、最大字数、时长、停顿、英文边界、去重叠；所有引擎共用同一配置（如 `maxSubtitleLength`） |
| 文本质量增强 | AI 校对 | 在分段之后可选执行，失败不得毁掉已有字幕 |

不要回到「三种转写方式各自维护一套分词合并」的结构。

## 4. 交付策略

- 应用内编辑结果必须能落到文件系统
- 输出路径必须可理解、可预测
- 与剪辑软件的衔接应尽量通过标准文件格式完成

## 5. 暂不锁定的部分

- 不锁定未来 UI 框架细节
- 不锁定服务内部接口形状
- 不锁定当前脚本和目录监听实现方式
- 不锁定当前日志、模型下载、测试按钮的 UI 呈现方式
