# 104 技术选择文档

## 1. 技术选择目标

这里只记录对产品形态有影响的技术决策，不记录临时代码实现。

## 2. 已锁定的产品级选择

| 主题 | 选择 | 原因 |
|------|------|------|
| 运行平台 | macOS 桌面应用 | 目标用户明确，且要衔接 Final Cut Pro |
| 工作模式 | 本地优先 | 降低云依赖和等待成本 |
| 主输入 | 音频、SRT | 当前只处理音频转写和现成字幕编辑，不支持 MP4、MOV、MKV 等视频文件 |
| 主输出 | SRT、FCPXML | 满足通用字幕交付与 FCP 工作流 |
| 配置方式 | 本地持久化 | 单用户桌面工具更简单直接 |
| 智能增强 | AI 校对可选 | 提升质量但不阻塞主流程 |
| 自动化能力 | 目录监听可选 | 服务进阶工作流，不影响基础使用 |

## 3. 引擎策略

- 转写引擎支持本地与云端并存
- 默认优先本地方案
- 当本地不可用或质量不足时，允许切到云端
- AI 校对是增强能力，不应成为唯一通路

## 4. 交付策略

- 应用内编辑结果必须能落到文件系统
- 输出路径必须可理解、可预测
- 与剪辑软件的衔接应尽量通过标准文件格式完成

## 5. 暂不锁定的部分

- 不锁定未来 UI 框架细节
- 不锁定服务内部接口形状
- 不锁定当前脚本和目录监听实现方式
- 不锁定当前日志、模型下载、测试按钮的 UI 呈现方式
