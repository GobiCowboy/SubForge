# 101 产品需求文档

## 1. 产品定位

SubForge 是一个面向 macOS 的字幕工作台，服务于“拿到一段音频或视频后，尽快生成、校对、微调并导出字幕给剪辑软件使用”的场景。

它的重点不是做成完整时间轴软件，而是把以下路径做短、做稳、做清楚：

1. 导入素材
2. 生成字幕
3. 必要时做 AI 校对
4. 人工微调时间与文本
5. 导出 SRT / FCPXML
6. 在需要时接入 Final Cut Pro 工作流

## 2. 目标用户

- 需要快速给音频或视频生成字幕的个人创作者
- 以 Final Cut Pro 为主的剪辑用户
- 希望先自动生成，再人工精修时间点和文本的用户
- 偏好本地优先、配置可控、流程直接的 macOS 用户

## 3. 本期目标

本期只定义可重做的一版产品边界，不继承当前实现中的混乱结构。

P0 目标：

- 让“导入 -> 转写 -> 编辑 -> 导出”成为一条稳定主路径
- 设置与主流程分离，不挤占编辑工作区
- 字幕编辑聚焦文本、入点、出点三个核心对象
- 支持 SRT 与 FCPXML 两种交付结果
- 支持目录监听这种偏自动化工作流，但不反客为主

## 4. 非目标

- 不做复杂时间轴
- 不做多轨字幕
- 不做项目管理系统
- 不做批量任务中心
- 不做团队协作
- 不把日志、测试工具、模型管理塞进主编辑区
- 不把当前代码结构当成未来重做的约束

## 5. 核心设计原则

- 主流程优先：首页只回答“导入什么、下一步做什么、设置在哪里”
- 编辑优先：字幕编辑页只强调文本和时间点
- 配置后置：设置独立存在，需要时再打开
- 本地优先：能本地完成的流程优先本地完成
- 可选智能：AI 校对是增强项，不应阻塞主路径
- 输出导向：最终必须产出可用的字幕文件，而不是停留在应用内

## 6. P0 功能清单

| 编号 | 功能 | 说明 |
|------|------|------|
| 301 | 工作区入口与素材导入 | 打开文件、拖放文件、最近项目 |
| 302 | 字幕生成流水线 | 转写状态、进度反馈、失败提示 |
| 303 | AI 校对 | 对转写结果做可选文本修正 |
| 304 | 字幕编辑工作台 | 编辑文本、入点、出点，插入与合并 |
| 305 | 导出与交付 | 输出 SRT、FCPXML，并衔接 FCP |
| 306 | 设置中心 | 引擎、语言、样式、导出、监听配置 |
| 307 | 目录监听工作流 | 监听 FCP 导出目录并自动处理 |

## 7. 成功标准

- 用户能在首次进入时理解导入入口和设置入口
- 转写过程有明确状态，不会“卡住但无反馈”
- 字幕可逐条修改文本、入点、出点
- 空字幕不会干扰最终输出
- 导出结果能直接进入常见剪辑交付流程
- 文档已足够支持后续重做，不再依赖现有代码推断需求
