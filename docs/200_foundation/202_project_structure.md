# 202 项目结构文档

## 1. 当前仓库主要目录

| 路径 | 用途 |
|------|------|
| `Sources/App/` | 应用入口、全局状态与命令入口 |
| `Sources/Models/` | 配置、最近项目、字幕片段等数据模型 |
| `Sources/Services/` | 本地持久化与配置存取能力 |
| `Sources/Utilities/` | 解析、时间格式化等纯工具能力 |
| `Sources/Views/RootView.swift` | 主窗口根布局，只负责组装首页 / 流水线 / 编辑页 |
| `Sources/Views/Home/` | 首页与导入入口 |
| `Sources/Views/Editor/` | 字幕编辑工作台 |
| `Sources/Views/Settings/` | 设置中心及其分组页面 |
| `Sources/Views/Shared/` | 流水线进度、侧边栏、Toast 等共享视图 |
| `docs/` | 产品、基础、功能和项目记忆文档 |
| `script/` | 本地构建运行入口 |
| `dist/` | 构建产物暂存 |

## 2. 文档目录规范

| 目录 | 作用 |
|------|------|
| `docs/100_requirements_architecture/` | 产品需求、流程、架构、技术边界 |
| `docs/200_foundation/` | 环境、结构、数据、质量、发布、日志 |
| `docs/300_features/` | P0 功能文档 |
| `docs/900_project_memory/` | 项目索引、问题和开发检查表 |

## 3. 文档摆放规则

- 产品目标放 100，不混进实现细节
- 基础规则放 200，不与功能说明交叉复制
- 单个功能的边界、规则、验收放 300
- 过程记录、索引、问题追踪放 900

## 4. 本轮重做原则

- 先稳定文档结构，再重做实现
- 后续重构代码时，优先更新 300 与 900，不回退到零散 README 式记录

## 5. 源码摆放规则

- `App/` 只放应用入口、场景定义、全局状态模型，不放细碎页面实现
- `Models/` 只放稳定数据结构，不放 UI 逻辑
- `Services/` 只放持久化、文件、外部能力接入，不放页面排版
- `Utilities/` 只放纯工具函数与解析能力
- `Views/Home/`、`Views/Editor/`、`Views/Settings/` 按页面域拆分，不互相堆叠
- `Views/Shared/` 只放跨页面复用的视图，不把单页私有视图误放进去

## 6. 当前结构约束

- 首页、编辑页、设置页必须继续保持拆分，不允许重新回到单文件堆叠
- 设置页的每个主分区继续独立成文件，避免把所有配置重新塞回一个视图
- 如果后续首页继续扩展，侧边栏行、状态条 badge、卡片样式应继续抽象，而不是直接堆到 `HomeView.swift`
