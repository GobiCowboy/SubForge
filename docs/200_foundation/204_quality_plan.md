# 204 质量计划文档

## 1. 质量目标

本项目后续重做时，优先验证“功能是否真的跑通”，而不是先追求功能数量。

## 2. 验收原则

- 每个 P0 功能必须有单独验收标准
- 主路径必须通过真实文件验证
- 失败路径必须有可理解反馈
- 设置保存后必须能实际生效
- 导出结果必须在外部工具中可继续使用

## 3. P0 验收场景

### 3.1 导入

- 可打开音频 / SRT
- MP4、MOV、MKV 等视频文件会被拒绝并提示不支持
- 拖放导入可用
- 最近项目可回到任务

### 3.2 转写与分段

- 有进度反馈
- 成功后进入编辑
- 失败时用户知道原因
- 三种引擎均经公共分段器产出字幕，切换引擎后最大字数等规则一致
- 有词级时间时不硬切英文单词；无词级时间时估算路径仍可用
- 引擎适配故障（如无效 URL）不崩溃，返回可读错误

### 3.3 校对

- 开启后自动参与流程
- 关闭后不影响主路径
- 失败时保留原始转写结果

### 3.4 编辑

- 可改文本、入点、出点
- 插入和合并逻辑清楚
- 空字幕不会让界面失控

### 3.5 导出

- 可得到 SRT
- 可得到 FCPXML
- 输出路径明确

### 3.6 设置

- 保存后能实际生效
- 配置可验证

### 3.7 监听

- 能发现新文件
- 能区分自动完成与人工校对两种模式

### 3.8 智能字幕与购买

- StoreKit商品不可用时显示可恢复错误，不展示硬编码价格。
- 恢复或重复提交购买不重复发放智能秒数。
- 官方API Key只存Keychain，日志不打印Key、上传签名、完整`oss://`或字幕正文。
- 智能任务首版只发送`processingRegion=china`。
- 音频请求目标必须是Model API签发的阿里`upload_host`，不上传到Billing或Model API。
- 任务成功按服务端`actualSeconds`刷新余额；失败显示返还结果。
- 本地转写、自定义云Provider和既有导出流程不因官方服务不可用而阻塞。

## 4. 文档检查清单

- 功能文档是否覆盖所有 P0 功能
- 功能边界是否互不冲突
- 是否把实现细节误写成产品规则

## 5. 后续测试建议

- 先做人工冒烟，再补自动化测试
- 优先覆盖导入、转写、编辑、导出主链路
