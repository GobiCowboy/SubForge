# 302 字幕生成流水线

## 1. 功能目标

把原始素材转成可编辑的字幕片段，并在过程中提供明确反馈。

## 2. 主要场景

- 用户导入音频后等待生成字幕
- 用户想知道当前是准备中、转写中还是已完成
- 用户遇到失败时需要立即知道原因

## 3. 功能范围

包含：

- 文件准备
- 调用转写引擎（Apple 语音 / 本地 Whisper / 云端 ASR）
- 展示阶段状态
- 展示总体进度
- 失败提示
- 成功进入编辑页

不包含：

- 转写引擎内部实现
- 批量转写调度

## 4. 关键规则

- 流水线状态必须肉眼可见
- 失败提示必须可理解，不能只有“失败”
- 转写完成后要产出可继续编辑的字幕片段
- 若开启 AI 校对，转写完成后自动进入下一阶段
- 本地 Whisper 缺少模型时，应能在设置中看到下载入口
- Whisper 模型下载应优先支持国内可访问镜像

## 5. 用户反馈要求

- 当前步骤
- 当前使用的引擎
- 是否可取消
- 完成 / 失败结果
- 设置页中必须能用内置测试音频验证当前转写配置

## 6. 验收标准

- 导入音频后可进入转写态
- 过程有阶段性反馈
- 成功后生成字幕并进入编辑
- 失败时用户能知道下一步该检查什么
