# 302 字幕生成流水线

## 1. 功能目标

把原始素材转成可编辑的字幕片段，并在过程中提供明确反馈。

流水线在产品边界上拆成两段，而不是三种引擎各自端到端切句：

1. **转写**：引擎把音频变成带时间的原始词元（或粗段落）
2. **分段**：公共分段器把词元切成可编辑字幕片段

## 2. 主要场景

- 用户导入音频后等待生成字幕
- 用户想知道当前是准备中、转写中还是已完成
- 用户遇到失败时需要立即知道原因
- 用户切换 Apple / Whisper / 云端 ASR 后，字幕切分规则保持一致

## 3. 功能范围

包含：

- 文件准备
- 调用转写引擎（Apple 语音 / 本地 Whisper / 云端 ASR）
- 引擎结果归一为带时间词元
- 公共分段（标点、字数、时长、停顿、英文边界、去重叠）
- 展示阶段状态与总体进度
- 失败提示
- 成功进入编辑页

不包含：

- 各厂商 SDK / CLI 的内部实现细节
- 批量转写调度
- 人工编辑页里的插入 / 合并（见 304）

## 4. 架构边界（必须遵守）

```
音频
  ├─ AppleSpeechProvider      ─┐
  ├─ WhisperCppProvider       ─┼─→ [SubtitleWord] 或粗 segment
  └─ CloudASRProvider         ─┘
                │
                ▼
     TimedSubtitleSegmenter（公共分段器）
                │
                ▼
          [SubtitleSegment]  → 可选校对 → 编辑
```

| 阶段 | 负责 | 不负责 |
|------|------|--------|
| 引擎适配 | 授权、调用、协议、原始时间戳解析 | 按产品规则切最终字幕 |
| 词元归一 | 输出 `SubtitleWord`；无词时间则保留粗 segment 供估算 | 三套不同的字数 / 标点策略 |
| 公共分段 | 强/软标点、最大字数、最大/优选时长、停顿间隙、英文不拆词、去重叠 | 网络、证书、模型下载 |

### 4.1 各引擎在「转写」侧的职责

- **Apple 语音**：拿到系统 segment，做异常超长 duration 收敛，映射为词元后交给公共分段器
- **本地 Whisper**：跑 whisper-cli，解析 JSON（含 DTW 词级时间）；有词元走真实边界，无词元走估算路径
- **云端 ASR**：兼容模式 / DashScope 异步任务等协议细节留在云端适配器；有 `words` 用真实词时间，否则估算
- **DashScope filetrans**：本地音频先上传至百炼临时存储（`oss://` 短链，约 48 小时），再提交异步转写；禁止整文件 Base64 塞进请求体（会触发 HTTP 413 RequestTooLarge）

### 4.2 公共分段规则

- 配置来源统一：如 `maxSubtitleLength`（默认 24，钳制在合理范围），作用于**所有**引擎
- 强标点优先断句；软标点在接近字数/时长阈值时断句
- 超过最大字数或最大时长必须切分；词间过长停顿应开新字幕
- 不得拆开英文单词
- 相邻字幕时间不得重叠
- 本地 Whisper 优先真实词边界；无词级时间时允许估算，但估算后仍走同一分段器

## 5. 关键规则

- 流水线状态必须肉眼可见
- 失败提示必须可理解，不能只有「失败」
- 转写 + 分段完成后要产出可继续编辑的字幕片段
- 若开启 AI 校对，分段完成后自动进入下一阶段
- 本地 Whisper 缺少模型时，应能在设置中看到下载入口
- Whisper 模型下载应优先支持国内可访问镜像
- 开发版与 Developer ID 包优先使用 Metal，GPU 路径不可用时自动回退 CPU
- 无效云端 ASR URL 不得崩溃，应返回可读配置错误

## 6. 用户反馈要求

- 当前步骤
- 当前使用的引擎
- 是否可取消
- 完成 / 失败结果
- 设置页中必须能用内置测试音频验证当前转写配置（验证链路也应经过同一公共分段逻辑）

## 7. 验收标准

- 导入音频后可进入转写态
- 过程有阶段性反馈
- 成功后生成字幕并进入编辑
- 失败时用户能知道下一步该检查什么
- 切换三种引擎后，单条字幕最大字数等分段参数表现一致
- 有词级时间的结果不会被硬切到英文单词中间
- 无词级时间的接口仍能产出合理字幕（估算路径），而不是各引擎各切一套
