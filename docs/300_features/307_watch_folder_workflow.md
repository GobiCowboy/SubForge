# 307 目录监听工作流

## 1. 功能目标

为 Final Cut Pro 工作流提供一个“导出音频后自动进入字幕处理”的旁路能力，减少从 FCP 到 SubForge 的重复导入操作。

## 2. 主要场景

- 用户在设置中选择监听目录并开启监听
- FCP 将音频导出到该目录
- 系统发现稳定的新音频，并确认来源是 Final Cut Pro
- SubForge 自动前台显示，并进入现有转写、校对、字幕编辑流程

## 3. 功能范围

包含：

- 选择监听目录
- 启动 / 停止监听
- 发现 FCP 导出的新音频素材
- 触发转写与可选校对
- 载入现有处理流水线并进入编辑器
- 通知处理状态

不包含：

- 大规模队列调度
- 后台服务常驻管理
- 团队共享监听任务
- 非 FCP 来源音频的自动处理

## 4. 关键规则

- 监听必须可显式开启和停止
- 监听不是默认主流程
- 应避免重复处理同一素材，启动监听时不处理目录中已存在的旧文件
- 监听目录作为信任边界，新稳定音频都会进入处理；Spotlight 的 Final Cut Pro 元数据只用于诊断日志，不作为硬性门槛
- 新文件按 0.5 秒间隔采样，修改时间和文件大小连续一致后才处理，避免导出未完成时提前接入
- 启动监听时只记录启动时间，不在主线程全量深扫历史文件；扫描会跳过 FCP 库、ScreenStudio 工程和其他包内容，避免历史素材目录拖慢监听
- 如果当前已有转写任务在跑，新文件保持等待状态，下次扫描再尝试接入
- 监听到可处理文件后，应用需要前台显示并走统一的 `importDocument(at:)` 入口

## 5. 结果模式

### 5.1 当前处理模式

- 自动进入转写
- 按现有设置决定是否校对
- 转写/校对完成后进入字幕编辑器

### 5.2 后续可扩展

- 监听队列
- 自动导出 FCPXML
- 导出后自动导入 Final Cut Pro

## 6. 验收标准

- 能检测到 FCP 导出的新音频
- 不会对同一文件无限重复处理
- 发现新文件后应用能前台显示
- 新文件能进入现有转写、校对、编辑链路
- 用户能看到当前处理状态
