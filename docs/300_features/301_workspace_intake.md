# 301 工作区入口与素材导入

## 1. 功能目标

让用户在进入应用后的 10 秒内知道三件事：

- 现在可以导入什么
- 导入后会发生什么
- 设置入口在哪里

## 2. 主要场景

- 打开一个新的音频文件开始转写
- 拖入一个现成 SRT 直接修改
- 从最近项目重新进入上次处理过的素材

## 3. 功能范围

包含：

- 点击打开文件
- 拖放导入
- 根据文件类型路由到转写或编辑
- 最近项目列表
- 底部默认配置摘要
- 设置入口

不包含：

- 批量导入
- 项目库管理
- 复杂素材筛选

## 4. 关键规则

- SRT 导入后应直接进入编辑
- 音频导入后进入生成流水线
- 不支持 MP4、MOV、MKV 等视频文件导入
- 最近项目只承担快捷入口，不承担项目管理
- 首页信息密度要低，避免功能堆叠

## 5. 页面要求

- 左侧或主视觉区域必须有明确导入动作
- 左侧侧边栏承担主导航与设置入口
- 最近项目单独成区
- 设置入口清楚但不喧宾夺主，应位于侧边栏底部
- 用户能看到当前默认引擎、语言、校对状态与监听状态摘要
- 首页视觉层级应保持简单，避免把项目管理、模板库、归档库等复杂入口堆回首页

## 6. 验收标准

- 用户能通过点击或拖入成功开始任务
- 文件类型分流正确
- 最近项目能重新打开仍然存在的文件
- 不存在的最近项目能被识别并清理
